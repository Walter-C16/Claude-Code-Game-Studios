## Part B — Proposed 0.0.1 scope

**Goal**: 30–45 min playable slice on Web + Windows. Prove the core loop (combat + dialogue + romance hook) with one complete emotional arc.

1. **Chapter 1 only**, splash → `ch01_complete`. No Ch2 hooks.
2. **3 characters visible**: Artemis (primary romance, 90% of player contact), Hippolyta (combat rival introduction), Priestess (lore NPC, gives the tutorial blessing). **Hide the other 11 entirely** — gallery shows ??? for unmet companions.
3. **4 combat encounters**: forest_monster (tutorial, unloseable), mountain_beast (mid), gaia_spirit (boss), + tavern tournament optional. No Abyss mode.
4. **Artemis romance arc through stage 2** — enough to show gift flow and trust gain, not full CG payoff.
5. **Camp loop exposed once** — post-first-dialogue, a "talk + gift Artemis" cycle so players see the loop. No mid-chapter camp grinding required.
6. **Divine Blessings introduced once** — Priestess hands one to the party before mountain_beast so the mechanic has an on-ramp.
7. **Hub stripped** to: Continue, Chapter Map, Camp, Settings, Gallery. Abyss/Tavern/Equipment gated behind a "Coming in 0.0.2" tooltip or fully disabled.
8. **Save/Continue from splash**, single save slot. Multi-slot (LAUNCH-005) deferred.
9. **First-run tutorials**: 3 overlays (chapter map, combat hand submit, camp talk). Reuse existing `TutorialOverlay`.
10. **End-of-demo screen**: "Thanks for playing — wishlist + Patreon" card on `ch01_complete`, then return to main menu.
11. **Settings**: text speed, music/SFX volume, locale (en only for 0.0.1).
12. **Builds**: Web HTML5 (primary — itch.io embed) + Windows Desktop (fallback).

Everything else — Nyx, Atenea, the 7 quest companions, Abyss mode, equipment depth, exploration missions, achievements, Spanish localization — is **0.0.2+ content**.

## Part D — Critical regression risk areas (from QA-lead scan)

Post gacha+epithet removal, these are the seams to sanity-test **first**:

1. **Save/load round-trip**. `game_store_serialization_test.gd` asserts exactly 22 top-level keys. Test: write a save on the current build → quit → relaunch → confirm no null-ref. Then test loading a pre-cleanup save — expect the 7 removed fields to be silently ignored, not crash.

2. **Blessing slot 4**. Used to have two unlock paths (Epithet III OR romance stage 3). Now only stage 3. Confirm slot 4 is reachable via a full romance path and doesn't stay perma-locked.

3. **`meet_companion` auto-add to deck**. New Game → progress to first companion meet → verify she lands in deck → save/quit → reload → verify she's still in deck. This path is where gacha hooks used to live.

4. **Tavern tournament chain**. `_launch_next_tavern_round` relaunches COMBAT scene between rounds. Verify all 4 rounds complete without save corruption and rewards land only after round 4.

5. **No stale tutorial text**. `TUTORIAL_EQUIP_BODY` and `TUTORIAL_DECK_BODY` were cleaned in earlier commits but spot-check that no `TUTORIAL_*_BODY` string mentions Oracle/Forge/Bond Shards/Epithets.

## Part F — Minimum quality gates for 0.0.1

Not full-ship standards — just "don't embarrass us on itch.io":

- **Crash-free**: Game reaches Hub from New Game without error dialog. Save → quit → relaunch succeeds. No first-minute crashes.
- **Core loop completable**: Player can meet Artemis → enter combat → win or lose → return to Hub. Tavern tournament does not softlock mid-run.
- **Data integrity**: Companions unlock, `companion_levels` persists across save/load, save file doesn't erase progress on load.
- **First impression**: Splash → Hub → first combat reachable with no dev-only branch, no visible "TODO" strings, no tutorial referencing removed systems.

Out of scope for 0.0.1 QA: full save-migration matrix (no players had old saves), romance beyond stage 2, equipment depth, Abyss mode, locale sweep, Android low-end perf, accessibility audit beyond existing tests, soak tests over 20 min.
